VigilAgent — Database Schema
Version: 3.0 | Date: June 2026 | Status: Final
1. Schema Design Principles
1. UUID Primary Keys — All entities use UUIDs for distributed-safe, non-sequential IDs
2. Soft Deletes — Never hard-delete; use deleted_at timestamp for recoverability
3. Audit Columns — Every table has created_at, updated_at, deleted_at
4. JSONB for Flexibility — Use PostgreSQL JSONB for extensible metadata fields
5. Vector Columns — Embeddings stored alongside business data via pgvector
6. Partitioning Ready — High-volume tables designed for time-based partitioning

8. Migration Strategy
8.1 Migration Tool
Use golang-migrate/migrate for schema migrations:
# Create new migration
migrate create -ext sql -dir migrations -seq create_users_table
# Run migrations
migrate -path migrations -database "postgres://localhost:5432/vigilagent?
sslmode=disable" up
# Rollback
migrate -path migrations -database "postgres://localhost:5432/vigilagent?
sslmode=disable" down 1
8.2 Migration Naming Convention
000001_create_users_table.up.sql
000001_create_users_table.down.sql
000002_create_projects_table.up.sql
000002_create_projects_table.down.sql
8.3 Data Migration Strategy
1. Zero-Downtime Migrations — Use CREATE TABLE without locks, then ALTER TABLE 
with NOT VALID constraints
2. Backfill Data — Run data migrations as background jobs, not in migration files
3. Rollback Safety — Every migration must have a down migration that restores the previous 
state
4. Test Migrations — Run migrations against a copy of production data before deploying

9.3 Read Replicas
// Primary for writes, replicas for reads
type DBPool struct {
    primary  *pgxpool.Pool
    replicas []*pgxpool.Pool
    counter  uint64
}
func (p *DBPool) Read() *pgxpool.Pool {
    idx := atomic.AddUint64(&p.counter, 1) % uint64(len(p.replicas))
    return p.replicas[idx]
}
func (p *DBPool) Write() *pgxpool.Pool {
    return p.primary
}
