C3: Complexity Thresholds — Architecture vs. LLM Strategy

(0.2) + novelty (0.15) + risk (0.3)
Resolution needed: Adopt the LLM Strategy's explicit formula as the canonical implementation.
Update Architecture doc to reference it.
C4: Embedding Dimensions — Schema vs. LLM Strategy
Document Dimensions Context

Contradiction: Schema uses 1536 dimensions (OpenAI ada-002 / text-embedding-3-small), while LLM

vectors with 384-dim queries.

Resolution needed: Decide on a single embedding dimension. Options:

9. Summary
The 12 optimized documents are well-structured and comprehensive, with strong architectural
consistency. The main issues are:
1. Numerical inconsistencies in pricing, rate limits, and token budgets — mostly between the
Master doc and API Contract
2. Structural gaps in billing, deployment, and data retention — topics mentioned but not detailed
3. Embedding dimension mismatch between Schema (1536) and LLM Strategy (384 for Ollama)
4. Missing memory layer — Procedural Memory exists in Schema but not in
Architecture/Continuation
Recommended next step: Resolve the 8 contradictions (Priority 1) before beginning implementation,
as they affect the core data model and API contract.
